# Code of conduct

## Our standard

ResizeLint contributors are expected to make technical discussion welcoming, specific, and evidence-based. Treat different experience levels with respect, critique behavior and code rather than people, and assume good intent while still addressing concrete harm.

Examples of welcome behavior include giving actionable review feedback, acknowledging uncertainty, protecting private project information, and correcting mistakes without hostility.

Harassment, discriminatory language, personal attacks, unwanted disclosure of private information, and sustained disruption are not acceptable in project spaces.

## Enforcement

Maintainers may edit or remove contributions that violate this standard and may temporarily or permanently restrict participation. Reports should use the private contact path described in [SECURITY.md](SECURITY.md) when public discussion would expose personal or sensitive information.

Enforcement decisions should be proportionate, documented, and focused on restoring a safe technical environment.
