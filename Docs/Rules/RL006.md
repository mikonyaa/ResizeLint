# RL006 — idiom-layout-decision

Default severity: **warning**. Automatic fix: **none**.

## What it finds

RL006 reports `.phone` and `.pad` comparisons when they select dimensions, constraints, columns, navigation, sidebars, tabs, modal presentation, or other layout behavior.

```swift
let columns = UIDevice.current.userInterfaceIdiom == .pad ? 4 : 2
```

## Adaptive alternative

Use the space that is actually available:

```swift
func columnCount(for width: CGFloat) -> Int {
    switch width {
    case ..<520: 2
    case ..<820: 3
    default: 4
    }
}
```

Size classes are appropriate when the design maps to semantic UIKit traits. Capability checks and protocol metadata that do not drive layout are excluded.
