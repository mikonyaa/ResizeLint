# RL009 — fullscreen-requirement-review

Default severity: **info**. Automatic fix: **none**.

## What it finds

RL009 reports a full-screen requirement in a property list or relevant build setting.

```xml
<key>UIRequiresFullScreen</key>
<true/>
```

```text
INFOPLIST_KEY_UIRequiresFullScreen = YES;
```

## Review guidance

The setting is not treated as a complete opt-out from resizing. Confirm that discrete resizing and the supported orientations match the product's real requirements.

ResizeLint does not remove the key automatically. Full-screen behavior can be deliberate for games, media, or specialized interaction, so this rule remains informational and supports a documented file-level override.
