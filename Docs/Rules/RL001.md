# RL001 — main-screen-bounds

Default severity: **error**. Automatic fix: **none**.

## What it finds

RL001 reports `UIScreen.main.bounds`, `nativeBounds`, and bounds reached through the main screen's coordinate spaces.

```swift
let pageSize = UIScreen.main.bounds.size
let pixels = UIScreen.main.nativeBounds
```

## Why it matters

The physical main display does not describe the available size of a view inside a resizable scene. The same process can own multiple windows with different geometry.

## Adaptive alternatives

Use geometry owned by the component that performs layout:

```swift
final class GalleryViewController: UIViewController {
    override func viewDidLayoutSubviews() {
        super.viewDidLayoutSubviews()
        let pageSize = view.bounds.size
        collectionView.frame = view.bounds
    }
}
```

When the requirement is explicitly scene-level, pass a `UIWindowScene` or its effective geometry from the caller. ResizeLint does not guess which replacement is architecturally correct, so RL001 has no automatic fix.
