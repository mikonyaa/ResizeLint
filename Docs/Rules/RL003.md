# RL003 — main-screen-reference

Default severity: **warning**. Automatic fix: **none**.

## What it finds

RL003 catches remaining `UIScreen.main` references not already covered by RL001 or RL002.

```swift
let screen = UIScreen.main
configurePresentation(for: UIScreen.main)
```

## Why it matters

A global screen reference hides which scene owns the operation. That ambiguity becomes visible when windows have different sizes, scales, or displays.

## Adaptive alternative

Pass the correct screen or geometry context from a view, window, or scene:

```swift
func configurePresentation(for screen: UIScreen) { /* ... */ }

if let screen = view.window?.windowScene?.screen {
    configurePresentation(for: screen)
}
```

Comments, string literals, suppressed ranges, and the more specific RL001/RL002 expressions do not produce RL003.
